1. ¿Qué problema concreto resuelve TypeScript que JavaScript no resuelve? Usa un ejemplo del restaurante que construyeron, no uno genérico.
Evita malformaciones, como cuando mi servicio devuelve datos que no empatan con un PlatoResponseDto. TS lo detecta de inmediato.

2. En el proyecto Node.js tienen models/Plato.js con un schema de Mongoose. Hoy crearon una interface Plato en TypeScript. ¿Por qué necesitan las dos? ¿No hacen lo mismo?
Tienen funciones distintas: TS previene bugs lógico-visuales en el editor, y Mongoose valida datos previos a MongoDB.

3. ¿Qué diferencia hay entre Plato y CreatePlatoDto? ¿Por qué el cliente no envía _id cuando crea un plato nuevo?
Plato tiene el registro entero con `_id`. CreatePlatoDto omite la base de datos, porque es el input limpio inicial del usuario.

4. Si mañana agregan una entidad Pedido con campos mesa, items y total, ¿qué crearían primero antes de tocar cualquier archivo del proyecto? Describe los pasos sin escribir código.
Estructurar sus tipos de datos, generando la interfaz Pedido y sus DTOs para entrada/salida.
